# 🎯 策略優化指南

## 📊 當前問題診斷

如果你的回測結果顯示:

```
總交易數: 204
勝率: 35.29%
總報酬: +0.11%
Profit Factor: 1.01
回測期間: 6個月 (180天)
```

**診斷**:
- ✅ 交易频率 OK: 204/180 ≈ 1.13 筆/天
- ❌ 勝率偏低: 35% << 45% (目標)
- ❌ Profit Factor 太低: 1.01 ≈ 打平
- ❌ 報酬過低: 0.11% / 180天 = 每天 0.0006%

**結論**: 不需要重新訓練模型,需要優化進場策略。

---

## 🛠️ 優化步驟

### 第 1 步: 提高機率閾值 (最重要!)

#### 原理

低閾值會導致:
- 接受太多低質量信號
- 勝率下降
- Profit Factor 下降

#### 操作

在 GUI 回測標籤:

```
當前設定:
Long 機率閾值:  0.16
Short 機率閾值: 0.16

建議調整:
Long 機率閾值:  0.20  (+25%)
Short 機率閾值: 0.20  (+25%)
```

#### 預期效果

| 指標 | 當前 (0.16) | 調整後 (0.20) | 改善 |
|------|--------------|-----------------|------|
| 交易數 | 204 | 120-150 | -30% |
| 勝率 | 35% | 42-48% | +20-37% |
| Profit Factor | 1.01 | 1.3-1.6 | +29-58% |
| 總報酬 (180天) | 0.11% | 3-6% | +25-50x |

---

### 第 2 步: 加入槓桿

#### 新墟能!

GUI 回測標籤現在支援槓桿設定。

#### 使用方式

```
回測設定:
│
├─ 回測天數: 180
└─ 槓桿倍數: [1, 2, 3, 5, 10]
         │
         ├─ 1x: 無槓桿 (預設)
         ├─ 2x: 適合新手
         ├─ 3x: 適合中手
         ├─ 5x: 適合熟手
         └─ 10x: 高風險!
```

#### 報酬計算

```
無槓桿報酬: +0.11%

2x 槓桿: +0.22%
3x 槓桿: +0.33%
5x 槓桿: +0.55%
10x 槓桿: +1.10%

注意: 槓桿同時放大風險!
```

#### 建議組合

| 策略 | 閾值 | 槓桿 | 預期年化報酬 | 風險等級 |
|------|------|------|----------------|--------|
| 保守 | 0.22 | 2x | 10-15% | 低 |
| 均衡 | 0.20 | 3x | 15-25% | 中 |
| 積極 | 0.18 | 5x | 25-40% | 中高 |
| 激進 | 0.16 | 10x | 40-60% | 高 |

---

### 第 3 步: 調整 TP/SL 比例

#### 當前設定

```
TP: 2.0%
SL: 1.0%
TP:SL = 2:1
```

#### 問題

- 如果勝率低,2:1 會導致更多停損
- 需要縮小 TP/SL 差距

#### 建議調整

根據勝率選擇:

| 勝率 | TP | SL | 比例 | 說明 |
|------|-----|-----|------|------|
| 35-40% | 1.5% | 1.0% | 1.5:1 | 縮小目標,提高勝率 |
| 40-45% | 1.8% | 1.0% | 1.8:1 | 均衡設定 |
| 45-50% | 2.0% | 1.0% | 2:1 | 維持現狀 |
| 50%+ | 2.5% | 1.0% | 2.5:1 | 擴大盈利 |

#### GUI 操作

```
資金管理:
│
├─ 停利 (%): 1.5  (從 2.0 調降)
└─ 停損 (%): 1.0  (維持)
```

---

### 第 4 步: 啟用自適應引擎

#### 為什麼使用自適應?

標準引擎的問題:
- 固定 TP/SL 不適應市場波動
- 沒有時段差異化
- 沒有機率分層

#### 操作

1. 選擇「進階自適應智能體」

2. 啟用功能:
   ```
   ☑ 波動率自適應  - 根據 ATR 調整 TP/SL
   ☑ 機率分層倉位  - 高機率加大倉位
   ☑ 時段差異化    - 不同時段不同策略
   ☑ 風控強化      - 日內虧損限制
   ```

3. 設定風控:
   ```
   最大日內虧損: 3%
   最大連續停損: 5 筆
   ```

#### 預期效果

```
標準引擎 vs 自適應引擎:

勝率:        35% → 42-48%  (+20-37%)
Profit Factor: 1.01 → 1.4-1.8  (+38-78%)
總報酬:      0.11% → 4-8%   (+35-70x)
```

---

### 第 5 步: 調整回測期間

#### 新墟能!

現在可以選擇回測天數:

```
回測天數: [7, 30, 60, 90, 180, 365]
```

#### 建議用法

| 期間 | 用途 | 優點 | 缺點 |
|------|------|------|------|
| 7 天 | 快速測試 | 快速回饋 | 樣本少 |
| 30 天 | 參數調整 | 平衡 | - |
| 60 天 | 策略驗證 | 覆蓋多種市況 | - |
| 90 天 | 完整季度 | 完整周期 | 較慢 |
| 180 天 | 預設 | 半年數據 | 較慢 |
| 365 天 | 完整年度 | 最完整 | 最慢 |

#### 建議流程

```
1. 快速測試 (30天)
   │
   ├─ 測試多個閾值: 0.16, 0.18, 0.20, 0.22
   ├─ 測試多個槓桿: 1x, 2x, 3x
   └─ 找到最佳組合
   
2. 中期驗證 (90天)
   │
   └─ 使用最佳組合驗證
   
3. 最終確認 (180天)
   │
   └─ 最終確認績效
```

## ⚠️ 常見錯誤

### 錯誤 1: 閾值太低

```
症狀:
- 交易數很多 (200+)
- 勝率很低 (< 40%)
- Profit Factor < 1.2

解決:
提高閾值 0.16 → 0.20 或更高
```

### 錯誤 2: 沒有使用槓桿

```
症狀:
- 勝率 OK (> 45%)
- Profit Factor OK (> 1.5)
- 但總報酬很低 (< 1%)

解決:
加入 2-3x 槓桿
```

### 錯誤 3: TP/SL 不合理

```
症狀:
- 勝率低但平均盈虧高
- 或勝率高但平均盈虧低

解決:
根據勝率調整 TP:SL 比例
```

### 錯誤 4: 不使用自適應

```
症狀:
- 標準引擎表現不穩定
- 不同時段表現差異大

解決:
切換到自適應引擎
```
