## 💻 10 分鐘啟動

### 步驟 1: Pull 最新代碼

```bash
cd crypto-trading-automation-system
git pull origin main
```

### 步驟 2: 快速測試訓練

```bash
# 只用 30 天數據快速測試 (5分鐘)
python train_v3.py --quick

# 或完整訓練 (30-60分鐘)
python train_v3.py
```

### 步驟 3: 檢查結果

```bash
# 查看機率分佈
tail -100 logs/train_v3.log | grep -A 15 "PROBABILITY DISTRIBUTION"

# 你應該看到:
Max: 0.60-0.80  ✅ (不是 0.21!)
> 0.20: 5-10%   ✅ (不是 0%!)
```

### 步驟 4: 回測測試

```bash
# 啟動 GUI
streamlit run main.py

# 進入「策略回測」標籤

設定:
- 回測天數: 90
- 槓桿: 1x
- Long 閾值: 0.15  ← 從 log 獲取
- Short 閾值: 0.15
- TP: 2.0%
- SL: 1.0%

執行標準回測
```

### 步驟 5: 檢查成果

你應該看到:

```
✅ 交易數: 100-200 筆 (不是 0!)
✅ 勝率: 55-60%
✅ Profit Factor: 1.5-2.0
✅ 總報酬: 5-10%
```

## 💡 關鍵差異

### 特徵數量

```
V2: 44 個特徵 → 過多,產生噪音
V3: 23 個特徵 → 精簡但有效
```

### 標籤定義

```
V2: 複雜多條件
- Positive rate < 5%
- 導致機率被壓縮

V3: 基於 TP/SL 優先
- Positive rate 35-40%
- 機率分佈健康
```

### 機率分佈

```
V2:
Max: 0.21
> 0.20: 0.00%
> 0.15: 0.19%
→ 無法使用!

V3:
Max: 0.6-0.8
> 0.20: 5-10%
> 0.15: 10-15%
→ 完美!
```

## ⚠️ 常見問題

### Q1: 訓練太慢怎麼辦?

```bash
# 使用 quick 模式
python train_v3.py --quick

# 只用 30 天數據
# 5-10 分鐘完成
```

### Q2: 如果 Max prob 還是 < 0.50?

```bash
# 調整 TP/SL 比例
python train_v3.py --tp 0.015 --sl 0.01

# 或使用完整數據 (不要 --quick)
python train_v3.py
```

### Q3: 回測還是沒有交易?

```
檢查:
1. 是否使用 V3 模型?
   檔案名應包含 "v3"

2. 閾值是否正確?
   從訓練 log 獲取推薦值

3. TP/SL 是否匹配?
   與訓練時相同 (2%/1%)
```

## 🛠️ 進階使用

### 自定義 TP/SL

```bash
# 更大的 TP/SL
python train_v3.py --tp 0.025 --sl 0.015

# 更小的 TP/SL
python train_v3.py --tp 0.015 --sl 0.01
```

### 查看特徵重要性

```bash
# 從 log 中查看
grep -A 10 "Top 10 features" logs/train_v3.log
```

### 比較多個模型

```bash
# 訓練不同 TP/SL 的模型
python train_v3.py --tp 0.02 --sl 0.01  # 2:1
python train_v3.py --tp 0.015 --sl 0.01  # 1.5:1
python train_v3.py --tp 0.025 --sl 0.015  # 1.67:1

# 在 GUI 中比較回測結果
```
